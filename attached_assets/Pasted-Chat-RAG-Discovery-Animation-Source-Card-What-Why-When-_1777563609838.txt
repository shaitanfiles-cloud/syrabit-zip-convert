Chat RAG Discovery Animation & Source Card
What & Why
When a user asks a question in chat, the AI currently shows a basic loading indicator while it resolves the answer. This task upgrades that into a step-by-step "thinking" animation that reveals exactly how the system found the answer — progressing from board syllabus search through topic, chapter, subject, class, and course — using real resolved values. After the response, a clickable source card is shown so the user can navigate directly to the matched topic and see exactly where the answer came from.

Done looks like
While the AI is thinking, the chat shows a sequential animated discovery trail:
"Searching Assam Board Syllabus..." (with a pulsing/scanning effect)
Topic matched → shows the actual resolved topic name (e.g. "Essence of Democracy")
Chapter resolved → shows the actual chapter name
Subject resolved → shows subject name
Class & course resolved → shows class + board (e.g. "Class 11 · AHSEC")
Each step fades/slides in only after the previous one, and shows the real dynamic value (not a placeholder)
The backend streams lightweight intermediate SSE discovery events during RAG resolution so each step triggers as it actually resolves, not all at once at the end
After the AI response, a source card appears beneath the message styled like the screenshot: chapter title in bold green, with board badge (e.g. "AHSEC"), class, and stream as pills
Clicking the source card navigates to the chapter/topic page using the existing URL pattern (board/class/stream/subject/chapter/topic/:slug)
On arrival at that page, the matched topic card briefly flashes green (a 2–3 second highlight animation) then returns to normal — reusing or extending the existing useHashScroll and TopicAnswerCard infrastructure
The source card only appears when the RAG source is library (not web or none)
The topic-to-keyword pipeline: the resolved topic/chapter/subject/class path is passed into the SEO keyword service so the generated keywords are scoped to the matched topic, not just the page
Out of scope
Redesigning the existing chat message bubble layout
Showing source cards for web-fallback answers
Multi-source cards (only the primary match is shown)
Changes to the quiz or highlight tools
Steps
Backend: stream intermediate discovery events — During RAG resolution in ai_chat.py, emit lightweight SSE events at each resolution milestone: discovery:searching (start), discovery:topic (topic name resolved), discovery:chapter (chapter matched), discovery:subject, discovery:class. Each event carries only the resolved value for that step. These fire inline during the existing resolve_rag_context call, before the final _meta_event.

Upgrade ThinkingIndicator — Rewrite ThinkingIndicator.jsx to listen for the new discovery:* SSE events and reveal each step sequentially with a fade-in/slide-up animation. Each step shows a label ("Topic", "Chapter", "Subject", "Class") and the resolved value as it arrives. The board-search pulse shows first immediately, then each subsequent step unlocks as its event arrives.

Frontend SSE parsing — Update the SSE stream handler in ChatPage.jsx to recognise and route the new discovery:* event types to the ThinkingIndicator state, without breaking the existing wai_chapter_match and _meta_event handling.

Source card component — Create a RagSourceCard component that renders the chapter title, board badge, class pill, and stream pill (matching the visual style shown in the attached screenshot — dark background, green title, styled pills). It receives props from the final _meta_event metadata fields (rag_chapter_name, ctx_board_name, ctx_class_name, ctx_stream_name, rag_chapter_slug, rag_topic_name).

Wire source card into chat — In ChatPage.jsx, after a message finishes streaming and the source is library, render the RagSourceCard below the assistant message bubble. Clicking it builds the correct deep-link URL and navigates using React Router.

Green highlight on arrival — In ChapterPage.jsx / TopicAnswerCard.jsx, detect a ?from=chat query parameter on arrival (appended by the source card link). When present, after the existing hash-scroll completes, apply a brief green background flash animation (CSS keyframe) to the matched TopicAnswerCard for 2–3 seconds, then fade back to normal.

Topic-scoped SEO keyword injection — In the backend, after RAG resolution produces a topic/chapter/subject/class match, pass that resolved path as context into SeoKeywordService so the generated meta keywords and AEO seed phrases are scoped to the specific topic, not just the page-level subject.
