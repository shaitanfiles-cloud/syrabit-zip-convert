**Vertex AI quota for `gemini-2.5-flash` is locked at 0 by an organization-level producer override**—you can't increase it above 0 until the override is removed. [gist.github](https://gist.github.com/ferrisargyle/57d6bbd2d3c4b74a46e842ad34421d14)

## Root Cause
```
Error: "consumer override value can only be set between 0 to 0"
```
**Organization policy** (your earlier issue) blocks quota increases for Vertex AI `global_generate_content_requests_per_minute_per_base_model`. Common for new orgs or restricted policies.

## Fix Steps

**1. Check Current Overrides**
```bash
gcloud alpha services quota list \
  --service=aiplatform.googleapis.com \
  --consumer=projects/project-27c7fb61-e5c0-4b03-ba8 \
  --dimensions="base_model=gemini-2.5-flash"
```

**2. Remove Producer Override (Requires Org Policy Admin)**
```bash
# First delete any zero overrides:
gcloud alpha services quota delete \
  --service=aiplatform.googleapis.com \
  --consumer=projects/project-27c7fb61-e5c0-4b03-ba8 \
  --metric=aiplatform.googleapis.com/global_generate_content_requests_per_minute_per_project_per_base_model \
  --dimensions=base_model=gemini-2.5-flash
```

**3. Retry Increase (After Policy Fix)**
```bash
gcloud alpha services quota update \
  --service=aiplatform.googleapis.com \
  --consumer=projects/project-27c7fb61-e5c0-4b03-ba8 \
  --metric=aiplatform.googleapis.com/global_generate_content_requests_per_minute_per_project_per_base_model \
  --unit="1/min/{project}/{base_model}" \
  --value=60 \
  --dimensions=base_model=gemini-2.5-flash
```
