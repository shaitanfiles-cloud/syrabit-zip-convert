To do what you’re trying to do via **shell / `gcloud`**, you need to:

1. Grant yourself or a principal the **Organization Policy Administrator** role at the **organization level**.  
2. Then use the console (or another `gcloud` command) to **change the `iam.managed.disableServiceAccountApiKeyCreation` policy** to **Not enforced** (which you can’t do purely via `gcloud` right now; it must be done in the console after you have the role).  

Below is the **exact shell path**.

***

### Step 1: Install & authenticate `gcloud`

Make sure you have:

- Google Cloud CLI installed.  
- Logged in as the user that owns / manages the org (`founder@syrabit.ai`):  

```bash
gcloud auth login
```

Then select your account.  

***

### Step 2: Find your Organization ID

Run:

```bash
gcloud organizations list
```

You’ll see output like:

```text
DISPLAY_NAME  ID  DIRECTORY_CUSTOMER_ID
syrabit.ai    123456789012  C1234567
```

Note the **ID** (e.g., `123456789012`); you’ll use it below.  

***

### Step 3: Grant `orgpolicy.policyAdmin` via shell

Run (replace `YOUR_ORG_ID` and `USER_EMAIL`):

```bash
gcloud organizations add-iam-policy-binding YOUR_ORG_ID \
  --member="user:founder@syrabit.ai" \
  --role="roles/orgpolicy.policyAdmin"
```

Example:

```bash
gcloud organizations add-iam-policy-binding 123456789012 \
  --member="user:founder@syrabit.ai" \
  --role="roles/orgpolicy.policyAdmin"
```

This:

- Gives `founder@syrabit.ai` the **Organization Policy Administrator** role on the org.  
- That role has all four required permissions:  
  - `orgpolicy.policy.get`  
  - `orgpolicy.policies.create`  
  - `orgpolicy.policies.delete`  
  - `orgpolicy.policies.update`. [docs.cloud.google](https://docs.cloud.google.com/iam/docs/roles-permissions/orgpolicy)

***

### Step 4: Use console to change the API‑key‑constraint

Now **in the browser**, go to:

- `https://console.cloud.google.com/iam-admin/orgpolicies`  
  - Make sure **Organization** `syrabit.ai` is selected.  
- Search for:  
  👉 `Block service account API key bindings`  
- Click it → **Manage policy**.  
- Change **Policy source** → **Override parent’s policy**  
- Set **Status** → **Not enforced**  
- Click **Set policy**. [docs.cloud.google](https://docs.cloud.google.com/organization-policy/apply-policies)

At this point, you’ll be able to proceed without the “permissions required” message.  

***

### If you want to revert later

To **remove** the role when you no longer need it:

```bash
gcloud organizations remove-iam-policy-binding 123456789012 \
  --member="user:founder@syrabit.ai" \
  --role="roles/orgpolicy.policyAdmin"
```

***

### If the `gcloud` command fails

If you get:

> `ERROR: (gcloud.organizations.add-iam-policy-binding) PERMISSION_DENIED`

then you or your account **cannot self‑grant** that role, and the **real org‑owner** must run it (or use the UI) for you. In that case, you can send them the above `gcloud` command and ask them to run it. [discuss.google](https://discuss.google.dev/t/cant-assign-organization-policy-administrator-role-to-myself/149617)
