Complete Workflow & Feature Inventory — Syrabit.ai
What & Why
A comprehensive inventory of every workflow, feature, and pipeline that powers Syrabit.ai — organized by functional area with status indicators. This serves as the master reference for understanding what exists, how it connects, and what needs attention.

1. Authentication & User Management
#	Feature	Description
1.1	User Signup	Email + password registration with free-tier auto-provisioning (30 credits)
1.2	User Login	JWT token generation with session + refresh cookies
1.3	Password Reset	Token-based reset flow with Resend email delivery
1.4	Auth Guard	Frontend route protection for authenticated pages (/chat, /profile, /history, /cms)
1.5	Admin Login	Separate admin authentication with AdminGuard route protection
1.6	Onboarding Flow	Post-signup onboarding page for new users to set academic preferences
1.7	User Profile	View/edit academic details, track credit usage, manage subscription
1.8	Session Management	Redis-backed sessions (TTL 1800s) with in-memory user cache (TTL 120s)
2. Content Library & Curriculum
#	Feature	Description
2.1	Library Browser	Browse subjects by Board (AHSEC, SEBA, DEGREE), Class, and Stream (Science/Commerce/Arts)
2.2	Subject Pages	Detailed subject view with Blog View (SEO articles) + Chapter Accordion (notes/summaries)
2.3	Curriculum Map	Visual/structured layout of the educational journey for students
2.4	Learn Pages (CMS)	SEO-optimized study guides at /learn/:slug
2.5	Subject Landing Pages	Programmatic SEO pages at /:board/:classSlug/:subjectSlug
2.6	SEO Topic Pages	Deep-topic pages at /:board/:classSlug/:subjectSlug/:topicSlug
2.7	PYQ Replica Pages	Previous Year Question replicas at /pyq/:slug
2.8	Exam Routine Page	Board exam schedule display at /exam-routine
3. AI Chat Pipeline (Syra)
#	Feature	Description
3.1	Non-Streaming Chat	POST /api/ai/chat — Full response AI chat with RAG
3.2	Streaming Chat (SSE)	POST /api/ai/chat/stream — Real-time token streaming with hybrid RAG
3.3	Query Classification	Classify queries as casual vs academic to route processing appropriately
3.4	RAG Search Order	Multi-tier retrieval: topic embeddings → chapter embeddings → subject content card embeddings
3.5	Exact Topic Match	When topic matches, retrieve from matched content card + lesson content (not generic web)
3.6	Web Search Fallback	DuckDuckGo/Serper fallback only after internal content yields insufficient context
3.7	Source Display	Show content card name + lesson name as source citations (not raw URLs/IDs)
3.8	Breadcrumb Source Trail	Display topic → chapter → subject path in chat UI alongside answers
3.9	Subject-Specific Context	Launch chat from subject page with auto-injected card context (subject details, chapters, tags)
3.10	Credit Deduction	1 credit per response, atomic deduction via Redis + PostgreSQL; refund on failure
3.11	Model Selector	UI dropdown to switch between AI models (GPT-4o, Sonnet, etc.)
3.12	Chat History	Conversation persistence and browsing at /history
3.13	Multi-Provider LLM	Groq (primary), Sarvam AI (fallback/translation), Google Vertex AI (embeddings)
4. Content Generation Pipeline (Admin)
#	Feature	Description
4.1	Lesson/Notes Generation	AI generates chapter notes (400-700 words) from chapter name + syllabus content → saved to MongoDB
4.2	Skip-If-Exists Check	Before generating notes, check if they already exist — skip if present
4.3	Mark-Wise Question Generation	Auto-generate 15 questions per chapter in 1, 2, 3, 5, and 10 mark tiers after notes exist
4.4	Flashcard / Memory Tricks	Generate mnemonics, shortcuts, mindmaps (text-based), and memory hacks per chapter
4.5	Atomic Chapter Update	After all generation completes, update chapter document atomically with notes + questions + flashcards
4.6	Bulk Subject Pipeline	1-click pipeline that processes all chapters in a subject sequentially (notes → questions → flashcards)
4.7	Pipeline Progress Panel	Real-time UI tracking of pipeline stages and per-chapter progress
4.8	Thumbnail Studio	AI-powered cover image generation using Vision analysis (Groq/Llama-4) with abstract color variants
5. SEO Engine & Programmatic SEO
#	Feature	Description

5.3	SEO Polish/Refinement	Rewrite, enrich, and deduplicate each generated SEO topic page; preserve all linkage
5.4	Blog Publish Wizard	Multi-step wizard: set canonical URL, publish date, JSON-LD structured data, breadcrumb linking (topic → chapter → subject)

5.9	15+ Prompt Variants	Multiple page types: Concept Notes, MCQs, Important Questions, etc.

21. Deployment — Cloudflare Pages + Railway (Split Architecture)
#	Feature	Description
21.1	Frontend on Cloudflare Pages	React SPA deployed to Cloudflare Pages — free unlimited bandwidth, global CDN, automatic SSL, DNS management
21.2	Backend on Railway	FastAPI backend deployed via Docker on Railway — proper long-running server with auto-scaling
21.3	Frontend/Backend Decoupling	Remove SPA-serving from backend (server.py static mount + index.html fallback); backend only handles /api routes
21.4	Cross-Origin API Calls	Frontend uses VITE_BACKEND_URL to call Railway-hosted backend (e.g. https://api.syrabit.ai); withCredentials for cookies
21.5	CORS & Cookie Config	SameSite=None + Secure cookies for cross-subdomain setup (syrabit.ai ↔ api.syrabit.ai); Cloudflare domain in allowed origins
21.6	Railway Deployment Config	railway.toml + Dockerfile pointing to backend; backend reads PORT from Railway environment
21.7	Cloudflare Pages Build	Build command: pnpm --filter @workspace/syrabit run build; output: artifacts/syrabit/dist; env: VITE_BACKEND_URL
21.8	Custom Domain Setup	syrabit.ai → Cloudflare Pages (frontend), api.syrabit.ai → Railway (backend)
21.9	Free CDN + SSL	Cloudflare provides free global CDN edge caching, automatic SSL certificate provisioning, and DDoS protection
21.10	Deployment Guide	Step-by-step instructions for both platforms: Cloudflare Pages setup, Railway Docker deploy, domain config, all environment variables
