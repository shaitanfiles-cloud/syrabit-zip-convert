Hybrid Render + Workers Deployment
What & Why
Deploy the Syrabit FastAPI backend to Render Starter ($7/mo) for always-hot hosting of heavy AI/RAG routes, and create a lightweight Cloudflare Worker as an edge proxy for caching, rate limiting, and simple utility routes. The frontend already deploys to Cloudflare Pages — this completes the production stack with zero Python rewrites.

This replaces the previously planned full-rewrite-to-Workers approach (tasks #133–#137) with a practical hybrid: FastAPI stays as-is on Render, Workers adds free edge caching/rate-limiting.

Done looks like
FastAPI backend runs on Render Starter with a render.yaml config, no cold starts
A Cloudflare Worker proxies and caches light routes (health checks, static responses, rate limiting) at the edge
Frontend API calls route to the correct backend based on endpoint type (heavy → Render, light → Workers)
CORS is configured correctly on both Render and the Worker for syrabit.ai
The split is transparent to users — all API functionality works as before
Out of scope
Rewriting any Python backend code to TypeScript/Workers
Changing the AI/RAG pipeline logic
MongoDB or Redis infrastructure changes
Frontend feature changes
Tasks
Render deployment config — Add render.yaml with the FastAPI service definition (Python runtime, start command, env vars, health check path). Ensure the backend starts correctly with Uvicorn on Render's expected port.

Cloudflare Worker edge proxy — Create a minimal Worker (wrangler.toml + entry script) that handles: health check responses, rate limiting via Workers KV or in-memory, caching of cacheable GET responses (subjects, content listings), and proxying uncached requests to the Render backend URL.

Frontend API route splitting — Update the API utility to define two base URLs (one for Render heavy routes like /api/ai/*, /api/admin/*, /api/razorpay/*; one for the Worker edge for light routes like /api/content/*, /api/health, /api/seo/*). Use environment variables (VITE_RENDER_API_URL, VITE_WORKER_API_URL) so the split is configurable per environment.

CORS and auth headers — Configure CORS on the Render FastAPI app to allow syrabit.ai and the Workers domain. Configure the Worker to forward auth headers (Bearer token, cookies, x-anon-id) to Render when proxying. Ensure withCredentials works across both origins.

Environment variables — Set up the required env vars for both Render (all existing backend secrets: MONGO_URL, JWT_SECRET, API keys, etc.) and Cloudflare Workers (RENDER_BACKEND_URL for proxy target). Update frontend env vars on Cloudflare Pages.
