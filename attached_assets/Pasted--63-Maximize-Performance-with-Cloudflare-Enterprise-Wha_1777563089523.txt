
#63 - Maximize Performance with Cloudflare Enterprise
What & Why
Syrabit is already on Cloudflare (Pages + Workers + KV + Vectorize), but several high-impact Enterprise features are unused. This task activates and wires up Load Balancing, Zaraz (Web Tag Management), Bulk Redirects, tiered caching, image optimization, and smart routing — removing bottlenecks at every layer from DNS to the browser.

Done looks like
Load Balancer routes API traffic between AWS App Runner and Railway with automatic health-check failover; no manual DNS edits needed during an origin outage
All third-party analytics/tag scripts (Google Analytics, Hotjar, etc.) run through Zaraz on the edge — zero render-blocking scripts reach the browser
All static redirect rules are moved from edge Worker JS to native Cloudflare Bulk Redirects (faster, zero Worker CPU cost)
Cloudflare Cache Rules cache API JSON responses and static assets aggressively with correct Cache-Control and Surrogate-Control headers
Polish (image compression) + Mirage (lazy loading) + Image Resizing are enabled and the imageCdn.js utility generates correctly-sized Cloudflare Image Resizing URLs
Argo Smart Routing is turned on for the api.syrabit.ai zone to reduce origin RTT
Tiered Caching (upper-tier POP) is enabled so cache misses go to a regional tier before hitting origin
HTTP/3 + QUIC and Early Hints (103) are enabled on both zones
Workers' edge-proxy is updated to set CF-Cache-Status bypass only for genuinely uncacheable routes (auth, POST), letting all read routes cache at edge
Out of scope
Changing backend application logic or database schema
Switching AI/LLM providers
Redesigning the frontend UI
Steps
Audit current cache and redirect configuration — Read workers/edge-proxy/wrangler.toml and the Worker source to catalogue which routes are bypassing cache, which redirects are handled in JS, and what response headers the FastAPI backend emits. Also check imageCdn.js for the current image URL format.

Migrate redirects to Cloudflare Bulk Redirects — Export all redirect rules currently handled in the Worker or any Cloudflare Page Rules into a JSON list suitable for the Bulk Redirects API. Remove the equivalent logic from the Worker to eliminate unnecessary CPU time.

Configure Load Balancing — Create a load balancer for api.syrabit.ai with two origins: the AWS App Runner endpoint and the Railway endpoint. Set HTTP health checks on /health (or /api/health), configure session affinity for WebSocket-style streaming responses, and set the failover policy so Railway is the fallback when App Runner is unhealthy.

Set up Zaraz for all third-party tags — Inventory every third-party script (analytics, error tracking, heatmaps, consent banners) loaded in the React app's index.html or injected by the Worker. Configure each as a Zaraz tool, enable Zaraz's built-in consent mode, and remove the <script> tags from the frontend bundle. Verify the Zaraz snippet is the only third-party HTTP request from the browser.

Write Cloudflare Cache Rules — Create fine-grained Cache Rules (replacing any legacy Page Rules) for: (a) /_next/static/* and /assets/* → cache 1 year with immutable, (b) GET /api/chapters/*, /api/syllabus/* → cache at edge for 5 minutes with stale-while-revalidate, (c) authenticated routes (/api/user/*, POST/PATCH/DELETE) → bypass cache. Update the Worker to pass Surrogate-Control headers that Cloudflare strips before forwarding to the browser.

Enable image optimisation — Turn on Polish (lossless + WebP conversion) and Mirage in the Cloudflare dashboard for the syrabit.ai zone. Update imageCdn.js to generate Image Resizing URLs (/cdn-cgi/image/width=…,format=auto/<original-path>) for every image component that currently requests a fixed-size URL, so every viewport gets the right size automatically.

Enable Argo, Tiered Cache, HTTP/3, and Early Hints — In the Cloudflare dashboard: enable Argo Smart Routing for both zones, enable Tiered Caching with a topology appropriate for India/Asia traffic (Singapore or Tokyo upper tier), turn on HTTP/3 with QUIC, and enable Early Hints (103) so Cloudflare can push Link: preload headers for critical fonts and the main JS chunk before the HTML is fully sent.

Update Worker to emit correct Vary and cache headers — Patch workers/edge-proxy to add Vary: Accept-Encoding, Accept on cacheable API responses, remove accidental no-store headers on read-only routes, and add Cache-Tag headers (e.g., chapter-{id}) to API responses so content can be purged by tag from the dashboard or CI pipeline after a publish event.

Validate and document — Run Lighthouse against key pages (home, chapter reader, quiz) and confirm Core Web Vitals improve. Document the new Cloudflare feature inventory (load balancer ID, Zaraz account ID, cache rule IDs) in a docs/cloudflare.md file so the team can maintain them.
