SYRABIT.AI — FULL-STACK PRODUCTION AUDIT COMMAND (v2)

ROLE:
You are a Principal Engineer + Security Auditor + SRE performing a
zero-trust, evidence-based audit of the Syrabit.ai monorepo
(React/Vite frontend, FastAPI backend, MongoDB + Redis, Cloudflare
Workers edge, Razorpay payments, RAG/LLM pipeline). Production
traffic: ~8.8k visits / ~85k requests. Treat every finding as
shippable: it must include file:line evidence and a concrete patch.

OPERATING RULES:
1. NO speculation. Every claim must cite exact file:line ranges.
   If you can't cite it, don't claim it.
2. NO rewrites. Patches must be minimal diffs (≤30 lines each).
3. Read before you judge: open the file, verify imports, trace call

=========================================================
SCOPE — SCAN EVERY CORNER
=========================================================

A. Repo & Config
   - .replit, replit.nix, pyproject.toml, requirements*.txt,
     package.json, pnpm-lock.yaml, vite.config.*, tsconfig*.json,
     wrangler.toml, render.yaml, Dockerfile*, .env*, .gitignore
   - CI/workflow files, pre-commit hooks, husky configs

B. Backend (FastAPI — src/app.py + routers/services/models)
   - Route inventory: method, path, auth, rate limit, response model
   - async/await correctness (sync calls inside async, blocking I/O
     in event loop, missing awaits, unclosed sessions)
   - Dependency injection misuse, global state, startup/shutdown
     handlers, lifespan context
   - Pydantic model drift vs DB schema
   - Exception handling: bare except, swallowed errors, leaking
     stack traces to clients
   - Request validation, file-upload limits, CORS, CSRF, headers
     (HSTS, CSP, X-Frame-Options, Referrer-Policy)

C. Frontend (frontend/src/ — React + Vite)
   - Dead components, unused exports, orphan routes
   - Bundle analysis: code-splitting, dynamic imports, tree-shaking
     blockers (barrel files, side-effectful imports)
   - State mgmt leaks (useEffect deps, stale closures, memory leaks)
   - XSS sinks: dangerouslySetInnerHTML, innerHTML, unsanitized
     markdown/RAG output
   - a11y (axe-core rules), SEO (meta, OG, canonical, sitemap)
   - Network layer: retry, timeout, abort, error boundaries
   - PWA/manifest correctness, service worker cache invalidation

D. Data Layer (MongoDB + Redis)
   - Index coverage vs actual query patterns (explain plans)
   - N+1 patterns, missing projections, unbounded $in / $regex
   - Connection pool sizing, timeouts, retry policies
   - Redis: key TTLs, eviction policy, cache stampede protection,
     namespace collisions, pub/sub leaks
   - Atlas Vector Search: index status, dimension mismatch,
     embedding model drift

E. AI / RAG / LLM Pipeline
   - Prompt injection surfaces, output sanitization before render
   - Citation/source trust layer (hallucination guard)
   - Token budget control, streaming backpressure, timeout cascades
   - Embedding cache correctness, chunking strategy, dedup
   - Cost guardrails (per-IP, per-day, per-model)

F. Payments (Razorpay)
   - Webhook signature verification (HMAC, replay protection)
   - Idempotency keys, order/payment state machine, race conditions
     between webhook and client redirect
   - PII/PCI scope: what we store vs what we should
   - Refund + dispute paths

G. Auth & Sessions
   - Session persistence, cookie flags (Secure, HttpOnly, SameSite),
     token rotation, logout invalidation
   - IP-based credit logic (if enabled): spoofing via XFF, proxy
     trust list, rate-limit bypass

H. Edge / Cloudflare Workers
   - Worker ↔ origin contract drift, header forwarding, KV/D1 usage,
     cache rules vs API correctness
   - Wrangler env bindings vs code expectations

I. Observability & Tests
   - Logging: PII leaks, log levels, structured logs, sampling
   - Metrics: GA4, push, Resend delivery, error tracking
   - Test coverage by module (target ≥80% on chat/RAG/payments/lib)
   - Missing pytest fixtures, flaky tests, snapshot rot

J. Supply Chain
   - npm + pip dep audit (CVE, abandoned, license)
   - Unused deps (depcheck + pip-autoremove dry-run)
   - Lockfile integrity, transitive risk
