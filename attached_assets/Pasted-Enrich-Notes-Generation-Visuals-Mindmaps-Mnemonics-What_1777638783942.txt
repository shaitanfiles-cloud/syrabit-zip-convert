Enrich Notes Generation: Visuals, Mindmaps, Mnemonics
What & Why
The current notes generator produces text-only outlines, key terms, and Q&A pairs. Students — especially those studying Commerce, Science, and Arts under the Assam Board (SEBA/AHSEC) — need visual study material: subject-specific tables (journal, ledger, final accounts), diagrams (cell structure, body systems, geographical maps), mindmaps, and memory-aid mnemonics. This upgrade makes the AI notes generator exam-oriented, visually rich, and board-specific.

Done looks like
Generated notes can include subject-aware tables rendered as proper HTML tables (journal entries, ledger accounts, trial balance, Trading/P&L, and Balance Sheet for Commerce; data tables for Science/Arts).
Generated notes can include diagram descriptions — structured ASCII-style or labelled-list representations of diagrams (cell organelles, plant/animal cell, human body systems, historical timelines, geographical features) that display clearly in the notebook.
Generated notes include a mindmap section when the topic has a clear hierarchical structure — rendered as an interactive or CSS tree in the UI.
Generated notes include a mnemonics section only when a strong, logically coherent mnemonic exists for the topic — the AI skips this section if no natural one fits, never forces it.
The prompt explicitly targets Assam Board (SEBA for Class 9/10, AHSEC for Class 11/12) exam patterns: past question styles, marks distribution language ("short answer", "long answer", "write in brief"), and Assam-specific examples where relevant.
Flashcard generation automatically picks up the new mnemonics field as extra cards.
All new sections are optional — notes for topics that don't need tables or diagrams remain clean and uncluttered.
Out of scope
Actual image generation or raster graphics — diagram content is structured text/ASCII, not images.
Chart rendering libraries (Chart.js etc.) — tables and structured lists cover the use case for this iteration.
Changes to the quiz generator or spaced-repetition algorithm.
Changes to source assemblers or RAG pipeline.
Steps
Extend the _NOTES_GEN_SYS prompt — Rewrite the system prompt in edu_study.py to:

Declare subject-aware generation rules: detect Commerce/Science/Arts from source content and emit appropriate tables/diagrams.
Add four new optional top-level JSON fields: tables, diagrams, mindmap, mnemonics.
Define exact schemas for each new field (see below).
Instruct the model to emit Assam Board-style language (SEBA/AHSEC marks patterns, "Write short notes on…", "Distinguish between…" Q&A formats).
Instruct the model to skip mnemonics and mindmap if the topic doesn't naturally support them — never force them.
New JSON field schemas to add to the prompt:

tables: array of { caption, headers: [str], rows: [[str]] } — for journal, ledger, final accounts, data tables.
diagrams: array of { label, type ("cell"|"flowchart"|"timeline"|"other"), content: str } — structured text description with labelled parts.
mindmap: { root: str, branches: [{ label: str, children: [str] }] } — hierarchical topic map; omit entirely if not logical.
mnemonics: array of { for: str, mnemonic: str, explanation: str } — only emit when naturally strong; omit array if none.
Update _validate_notes_payload and _coerce_notes_payload — Add validation, length clamping, and normalisation for the four new optional fields. Tables validate that headers and rows are string arrays of consistent width. Diagrams clamp content length. Mindmap validates branch/children structure. Mnemonics clamp to reasonable lengths. None of the new fields are required — the validator passes happily if they are absent or empty.

Update the persisted notes schema — The generated_body JSONB column already holds the full payload, so no DB migration is needed. Confirm the endpoint's response serialisation passes through the new fields untouched.

Render tables in NotebookPage.jsx — When a generated note's body includes tables, render each as a styled <table> with a caption row, sticky header, and alternating row shading. Commerce tables (journal/ledger) should use a monospace-friendly font for amount columns.

Render diagrams in NotebookPage.jsx — When diagrams are present, render each as a labelled card with a clear heading, a pre-style content block for structured text, and a subtle colour accent per diagram type (cell = green, flowchart = blue, timeline = amber, other = grey).

Render mindmaps in NotebookPage.jsx — When a mindmap is present, render it as a collapsible CSS/SVG radial or indented tree: the root node in the centre (or top), branches expanding outward with connecting lines, leaf nodes as chips. Keep it readable at mobile widths by defaulting to the indented/accordion view on small screens.

Render mnemonics in NotebookPage.jsx — When mnemonics are present, render each as a styled card showing the mnemonic phrase in large text, the topic it covers, and the explanation below. Use a warm highlight background to make them visually distinct.

Extend flashcard builder to include mnemonics — In the POST /api/edu/flashcards/build route (and studyApi.js), when building flashcards from a generated note, include one flashcard per mnemonic: front = "Mnemonic for [topic]?", back = mnemonic + explanation.
