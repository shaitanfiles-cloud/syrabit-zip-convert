=========================================================
SCOPE — CHECK EVERY SURFACE THAT GATES AI BOTS
=========================================================

1) robots.txt
   - Fetch: `curl -sI https://syrabit.ai/robots.txt` and
            `curl -s  https://syrabit.ai/robots.txt`
   - Verify: 200 OK, Content-Type: text/plain, ≤500KB,
            served from root (NOT /public/robots.txt redirect).
   - Parse every User-agent block. Flag if ANY of these are
     Disallowed (explicitly or via wildcard `User-agent: *`):
       GPTBot, OAI-SearchBot, ChatGPT-User, PerplexityBot,
       Perplexity-User, ClaudeBot, anthropic-ai, Claude-Web,
       Google-Extended, Applebot, Applebot-Extended, CCBot,
       Bytespider, Amazonbot, Meta-ExternalAgent,
       Meta-ExternalFetcher, DuckAssistBot, cohere-ai,
       YouBot, PhindBot, Diffbot, Timpibot, ImagesiftBot,
       Omgilibot, FriendlyCrawler
   - Check Sitemap: directive present and reachable.
   - Check for stray `Disallow: /` under `User-agent: *`.

2) llms.txt  +  llms-full.txt   (proposed standard)
   - Fetch root: /llms.txt and /llms-full.txt
   - Required structure:
       # Site Name
       > one-line value prop
       ## Docs / Products / About …
       - [Title](https://syrabit.ai/path): short description
   - Validate: UTF-8, markdown, ≤ a few hundred KB, links 200.

3) Cloudflare / Edge layer
   - Cloudflare dashboard has a "Block AI Bots" / "AI Scrapers
     and Crawlers" managed rule. Check if enabled.
   - WAF custom rules matching `cf.client.bot`,
     `http.user_agent contains "GPTBot"`, etc.
   - Bot Fight Mode / Super Bot Fight Mode setting.
   - Page Rules / Transform Rules rewriting robots.txt.
   - `wrangler.toml` + Workers code: any UA-based 403/451.

4) Origin server (FastAPI / Vite preview / Render)
   - Middleware blocking by UA (rg for: "GPTBot", "user-agent",
     "user_agent", "bot", "crawler", in middleware/, app.py).
   - Rate limiter rejecting first-touch requests with 429.
   - 5xx on /robots.txt or / for non-browser UAs.

5) Rendering & content accessibility
   - Is the homepage SSR/SSG, or pure CSR (Vite SPA)?
     AI crawlers vary: GPTBot DOES execute limited JS,
     PerplexityBot + ClaudeBot largely DO NOT.
   - Run: `curl -A "GPTBot/1.0" https://syrabit.ai/` and
          `curl -A "PerplexityBot" https://syrabit.ai/`
     Confirm non-empty <main>, real text content, not just
     `<div id="root"></div>`.
   - Check meta robots, X-Robots-Tag header for `noai`,
     `noimageai`, `noindex`.

6) Sitemap & canonical
   - /sitemap.xml: 200, lists key pages, lastmod fresh.
   - Each key page has <link rel="canonical">.
   - hreflang only if multilingual.

7) Structured data (boosts LLM citation)
   - JSON-LD on home + article/library pages:
     Organization, WebSite (with SearchAction), Article,
     FAQPage, BreadcrumbList.
   - Validate via schema.org rules; no required fields missing.

8) Server logs / analytics evidence
   - Grep last 30d access logs for the bot UAs above.
     Command: `rg -i "GPTBot|PerplexityBot|ClaudeBot|
                       Google-Extended|Applebot-Extended|
                       OAI-SearchBot|ChatGPT-User|CCBot"
                       <log path>`
   - Cloudflare Analytics → Security → Bots → "Verified bots".
   - If GA4 only: bots are filtered; check raw edge logs instead.

9) DNS / reachability
   - `dig syrabit.ai +short`, `dig www.syrabit.ai +short`
   - HTTPS cert valid, no MITM/redirect loops.
   - HTTP→HTTPS 301 (not 302), www↔apex consistent.

10) Discoverability signals OFF-site
   - Inbound links from indexed sites (Bing/Google index count).
   - Presence in Common Crawl latest dump (CCBot data feeds
     many LLMs). Query: https://index.commoncrawl.org/

=========================================================
EXECUTION COMMANDS (run in this order)
=========================================================

# A. Fetch + inspect
curl -sI https://syrabit.ai/robots.txt
curl -s  https://syrabit.ai/robots.txt
curl -sI https://syrabit.ai/llms.txt
curl -s  https://syrabit.ai/llms.txt
curl -sI https://syrabit.ai/sitemap.xml

# B. Bot-UA fetch tests
for UA in "GPTBot/1.0" "PerplexityBot/1.0" "ClaudeBot/1.0" \
          "Google-Extended" "Applebot-Extended" "CCBot/2.0" \
          "OAI-SearchBot/1.0" "ChatGPT-User/1.0"; do
  echo "=== $UA ==="
  curl -s -o /dev/null -w "%{http_code} %{size_download}\n" \
       -A "$UA" https://syrabit.ai/
done

# C. Header audit
curl -sI -A "GPTBot/1.0" https://syrabit.ai/ | \
  rg -i "x-robots-tag|cache-control|cf-|server"

# D. Repo scans
rg -n -i "gptbot|perplexitybot|claudebot|user-agent|x-robots-tag|noai" .
rg -n "robots\.txt|llms\.txt|sitemap" .
rg -n "allow_origins|cors|middleware" artifacts/syrabit/api/src
