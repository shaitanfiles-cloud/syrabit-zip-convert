voyage_embeddings: add one-shot disable on 401/403 to stop log spam Problem ------- The unfixable VOYAGE_API_KEY (HTTP 403) was producing ~945 identical WARNING log lines per syllabus-seeding pass — one per chapter and topic batch, every ~10 seconds at startup and on every reseed. The Gemini fallback was working correctly but Voyage was being retried wastefully on every call. Fix --- artifacts/syrabit-backend/voyage_embeddings.py - New module-level _DISABLED flag + _disable(reason) helper. First 401/403 triggers a single ERROR log explaining the cause and how to re-enable, after which is_configured() returns False and the embed pipeline skips Voyage entirely (straight to Gemini fallback). - Catches both raw response.status_code in (401, 403) and httpx.HTTPStatusError carrying those codes, so both code paths trip the breaker. - Auto-resets on process restart, so updating VOYAGE_API_KEY and restarting the api workflow re-enables Voyage cleanly without any code change. Verified live: zero voyage_embeddings warnings after restart; syllabus seeding pass completes silently; api health checks 200; no new errors introduced.
