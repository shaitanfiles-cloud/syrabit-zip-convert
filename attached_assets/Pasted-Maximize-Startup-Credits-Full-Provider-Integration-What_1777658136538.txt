Maximize Startup Credits — Full Provider Integration
What & Why
Integrate every startup-credit provider into Syrabit's provider stack via the Cloudflare AI Gateway BYOK system, and update llm.py's dispatch logic to follow the 80/20 rule: Workers AI handles 80% of each category for free; credit providers absorb the remaining 20% at peak load, saturation, or fallback. This maximises the value of ~$390k in confirmed startup credits (Google Cloud $200k, Microsoft for Startups $150k, Anthropic $25–50k, Groq $5–10k, Together AI $500, Fireworks AI $500, Mistral AI $1k, Perplexity $500, ElevenLabs $500, AssemblyAI $500, Voyage AI $500, Exa AI $1k, Tavily $500, AWS Activate $1k hosting-only) while keeping the marginal cost near zero at normal load.

Done looks like
Workers AI is the gated primary for chat, embeddings, reranking, translation, STT, vision, and safety — all free-tier calls.
Every credit provider is wired through CF AI Gateway BYOK so no raw API keys live in backend env vars.
llm.py select_provider / route_for_task applies load-based cascade: if Workers AI gateway metrics exceed 80% saturation, the next credit provider in the priority list is picked automatically.
Vertex AI / Gemini is restored as a primary fallback provider (position 2 for chat, using $200k Google Cloud Founders credits).
Together AI, Fireworks AI, Mistral AI (direct), and Perplexity are wired as additional LLM fallback slots.
ElevenLabs is wired for Indic-language TTS overflow (Hindi, Bengali, Assamese) where Cartesia quality is insufficient.
AssemblyAI is wired as an STT fallback for Indian-accented audio alongside Workers AI Whisper.
Voyage AI is wired for high-precision embedding fallback on long-form educational content.
Exa AI and Tavily are wired as neural search providers for live RAG queries (syllabus updates, exam notifications).
Sentry is integrated for error tracking; Axiom for unified log management across Railway, App Runner, and CF Workers.
AI4Bharat open models are documented as a free fallback for Assamese/Bengali when Sarvam AI quota runs out.
OpenRouter, Groq, Cerebras, Cohere, Pinecone AI, Cartesia, Deepgram, Baseten, Sarvam AI, and Replicate all have healthy BYOK paths confirmed via the gateway smoke test.
The admin health panel shows per-provider credit burn rate, remaining quota estimate, and a "credits low" warning card for any provider below 20% estimated remaining.
A nightly cron logs per-provider call counts to the existing audit digest so spend is visible without leaving the admin panel.
The admin credit burn panel includes an AWS hosting row showing estimated App Runner spend vs. the $1k AWS Activate grant (~$21/month → ~47 months of free hosting).
A CloudWatch billing alarm is set at $800 so the team is alerted before AWS credits run out.
Out of scope
AWS Bedrock or any AWS AI services — AWS $1k credits cover App Runner hosting only.
Purchasing or topping up credits (manual step, outside code).
Switching vector DB from Pinecone to another provider.
Any frontend chat UI changes.
Workers AI model upgrades beyond what is already in the SLM slot table.
