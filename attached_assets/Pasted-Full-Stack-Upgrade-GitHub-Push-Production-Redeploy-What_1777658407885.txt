Full-Stack Upgrade, GitHub Push & Production Redeploy
What & Why
Perform a complete, end-to-end production upgrade across all layers of the Syrabit.ai platform and migrate the Python FastAPI backend from Railway to AWS App Runner (via ECR). The migration path is:

BEFORE:  FastAPI → Railway → Cloudflare
AFTER:   FastAPI → ECR (Docker image) → App Runner → Cloudflare

Everything else stays exactly where it is — the Rust Axum core remains on Railway, and Cloudflare Pages, edge-proxy Worker, and email-worker are untouched as deployment targets. In addition to the infrastructure migration, every dependency across the full stack is upgraded to its latest stable release, FastAPI/Pydantic v2 patterns are applied consistently, and a clean GitHub push triggers the refreshed CI pipeline before production is promoted.

Done looks like
A Dockerfile exists in artifacts/syrabit-backend that produces a minimal, production-ready FastAPI image.
An ECR private repository (syrabit-backend) exists in the target AWS account and region.
GitHub Actions pushes a new image to ECR and triggers an App Runner redeploy on every merge to main.
The FastAPI backend is live on App Runner and /healthz/ai, /livez, /readyz all return 200 from the App Runner service URL.
The Cloudflare edge-proxy wrangler.toml origin URL is updated to point to the App Runner service URL instead of the Railway URL.
The Railway Python backend service is confirmed decommissioned (or left idle) after App Runner is verified healthy — no traffic reaches it.
Rust Axum core on Railway is redeployed on the latest crate versions and /health returns 200.
Every package.json, pnpm-workspace.yaml catalog, requirements.txt / pyproject.toml, and Cargo.toml is updated to the latest stable release with no unresolved peer warnings.
FastAPI and Pydantic v2 patterns are used consistently; all response models are typed; deprecated helpers are removed.
Cloudflare Worker compatibility dates are set to 2026-05-01 and Wrangler is on its latest release.
All changes are committed and pushed to main; all GitHub Actions CI workflows pass before production is promoted.
Cloudflare Pages frontend and both Workers are redeployed and smoke tests pass.
Admin health panel shows all services green (App Runner backend, Railway Rust core, Cloudflare Pages, edge-proxy, email-worker, AI Gateway, MongoDB, Supabase, Redis).
Out of scope
Migrating the Rust Axum core away from Railway.
Migrating Cloudflare Pages, edge-proxy Worker, or email-worker to any other provider.
Introducing new product features or UI changes.
Adding new API endpoints beyond what is already planned.
Setting up AWS infrastructure beyond ECR + App Runner + CloudWatch log group (no RDS, VPC, ALB, etc.).
Steps
Phase 1 — Dependency Audit & Upgrade
Python backend dependency upgrade — Inside artifacts/syrabit-backend, audit requirements.txt / pyproject.toml. Upgrade FastAPI, Pydantic v2, Uvicorn, httpx, motor, supabase-py, google-generativeai, groq, and all other first-party deps to their latest stable releases. Pin exact versions in the lockfile.

Rust Cargo upgrade — Inside backend/rust-core, run cargo update then cargo upgrade to bump all crates in Cargo.toml to their latest compatible semver. Run cargo audit and resolve any advisories. Confirm cargo build --release succeeds cleanly.

Node / pnpm catalog upgrade — Update the shared pnpm-workspace.yaml catalog: bump Vite (latest 7.x patch), Vitest, Playwright, Wrangler (4.x → latest), @supabase/supabase-js, @tanstack/react-query, lucide-react, framer-motion, Tailwind CSS v4, TypeScript (~5.9.x → latest), and all @radix-ui packages. Run pnpm install and resolve any peer conflicts.

Root security-pin override bump — Update the pnpm.overrides block in root package.json for undici, postcss, path-to-regexp, follow-redirects, lodash, and picomatch to their latest patched versions.

Worker-local dependency sync — Inside workers/edge-proxy and workers/email-worker, align local package.json Wrangler and TypeScript versions to match the catalog, delete stale package-lock.json files, and re-lock from the monorepo root.

Phase 2 — API & Backend Code Modernisation
FastAPI / Pydantic v2 sweep — Audit every router in artifacts/syrabit-backend. Replace any Pydantic v1 .dict() calls with .model_dump(), orm_mode with model_config = ConfigDict(from_attributes=True), and @validator with @field_validator. Add response_model= typing to any admin endpoint that is still untyped. Replace deprecated on_event startup/shutdown handlers with the lifespan context manager.

Health & readiness endpoint hardening — Ensure /livez returns app version, Git SHA, and uptime; /readyz checks MongoDB, Supabase, and Redis before returning 200; /healthz/ai pings the Cloudflare AI Gateway. Update health check documentation to reference the App Runner service URL instead of Railway.

Rust Axum route modernisation — Update any deprecated Axum 0.7→0.8 handler patterns (extract::State wiring, IntoResponse impls). Confirm gRPC reflection is enabled on port 50051 and the WebSocket /ws/metrics handler compiles and reconnects cleanly.

Cloudflare Worker compatibility date update — Set compatibility_date = "2026-05-01" in workers/edge-proxy/wrangler.toml and workers/email-worker/wrangler.toml. Review the Cloudflare changelog for breaking flags introduced since the previous date and fix any affected worker code.

Phase 3 — FastAPI Docker Image & AWS Infrastructure
Write the production Dockerfile — Create artifacts/syrabit-backend/Dockerfile. Use a multi-stage build: python:3.12-slim build stage installs dependencies from the lockfile, final stage copies only the app and the installed site-packages. Set CMD ["uvicorn", "server:app", "--host", "0.0.0.0", "--port", "8000"]. Add a .dockerignore that excludes __pycache__, *.pyc, .env, and test files. Verify the image builds locally and /livez responds.

Create ECR repository — Using the AWS CLI (or IaC script), create a private ECR repository named syrabit-backend in the target region. Enable image tag immutability and scan-on-push. Document the repository URI in a new artifacts/syrabit-backend/AWS-DEPLOY.md file.

Create CloudWatch log group — Create a CloudWatch log group /syrabit/backend with a 90-day retention policy. This is the centralised log target App Runner will stream to.

Create App Runner service — Create an App Runner service sourcing from the ECR image (syrabit-backend:latest). Configure: CPU 1 vCPU / Memory 2 GB (sufficient for FastAPI + asyncio workloads at this scale). Set the health check path to /livez. Inject all required environment variables (Supabase keys, MongoDB URI, CF AI Gateway URL and key, Upstash Redis URL, Cloudflare R2/KV/D1 tokens) as App Runner service environment variables or via AWS Secrets Manager references. Enable CloudWatch logging to /syrabit/backend. Document the resulting App Runner service URL.

Phase 4 — GitHub Actions CI/CD Update
Replace Railway deploy step with ECR + App Runner step — Update .github/workflows/railway-deploy.yml (or create a new aws-deploy.yml). The new job: authenticates to ECR via aws-actions/amazon-ecr-login, builds and pushes the Docker image tagged with the Git SHA and latest, then calls the App Runner start-deployment API to trigger a rolling redeploy. Add AWS_ACCESS_KEY_ID, AWS_SECRET_ACCESS_KEY, AWS_REGION, ECR_REGISTRY, and APP_RUNNER_SERVICE_ARN as GitHub Actions secrets.

Update GitHub Actions workflow action versions — In all .github/workflows/*.yml files, bump actions/checkout, actions/setup-python, actions/setup-node, actions/cache, and actions/setup-java (if present) to their latest major releases. Ensure edge-proxy-deploy.yml references the current Wrangler version.

Phase 5 — CI Green & GitHub Push
Run the full test suite — Execute pnpm --filter artifacts/syrabit test, pnpm --filter artifacts/syrabit build, backend pytest suite, and cargo test. Fix any failures caused by version bumps. Build the Docker image one final time to confirm the production build is clean.

Commit and push to GitHub — Stage all changes with the commit message: chore: dep upgrades + pydantic-v2 sweep + fastapi→ecr/apprunner migration. Push to main. Confirm all GitHub Actions CI workflows (frontend-tests, all-tests, backend-tests, the new aws-deploy) turn green.

Phase 6 — Cloudflare Redeployment
Update edge-proxy origin URL — In workers/edge-proxy/wrangler.toml (and any related env config), replace the Railway FastAPI origin URL with the App Runner service URL. Ensure the production and preview environments both point correctly.

Deploy edge-proxy Worker — Run the edge-proxy-deploy.yml pipeline. It deploys to preview, runs smoke tests from monitored-urls.json, waits for manual approval, then promotes to production. Confirm auto-rollback does not trigger.

Deploy email-worker — Run wrangler deploy from workers/email-worker. Verify all bindings (D1, KV, R2, secrets) resolve correctly in production.

Deploy frontend to Cloudflare Pages — Run pnpm --filter artifacts/syrabit build then trigger a Cloudflare Pages deployment. Confirm _routes.json and _headers are picked up and the production URL returns the React app with no CSP or CORS errors.

Phase 7 — Railway Rust Core Redeploy
Redeploy Rust Axum core on Railway — Trigger a new Railway deployment for the Rust core service from backend/rust-core/railway.toml. Confirm gRPC port 50051 and the HTTP REST port are both bound. Verify /health returns 200 and all /api/edge/d1-sync/* endpoints respond correctly.
Phase 8 — Post-Deploy Validation & Cleanup
Full smoke test sweep — Hit /livez, /readyz, /healthz/ai against the App Runner URL; hit /health, /api/edge/d1-status against the Railway Rust URL; hit all routes in monitored-urls.json via the Cloudflare edge. All must return expected status codes with no 5xx errors.

Admin health panel sign-off — Confirm the admin health panel shows green for: App Runner FastAPI backend, Railway Rust core, Cloudflare Pages, edge-proxy, email-worker, AI Gateway, MongoDB, Supabase, Redis.

Railway Python service decommission — Once App Runner is confirmed healthy and all smoke tests pass, stop or delete the Railway Python backend service to avoid double-billing. Update RAILWAY-DEPLOY.md to note it now covers Rust core only.

Update deployment documentation — Update artifacts/syrabit-backend/AWS-DEPLOY.md with the final App Runner service URL, ECR URI, and environment variable checklist. Update CLOUDFLARE_DEPLOYMENT_WIRING.md to reflect the new backend origin. Update PHASE3_DEPLOYMENT_GUIDE.md with the new architecture diagram line (FastAPI → ECR → App Runner → Cloudflare).
