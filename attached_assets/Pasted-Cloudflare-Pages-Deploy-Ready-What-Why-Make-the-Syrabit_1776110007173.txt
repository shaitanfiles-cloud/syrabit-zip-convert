Cloudflare Pages Deploy Ready
What & Why
Make the Syrabit frontend fully deployable on Cloudflare Pages. The app is a React+Vite SPA in a pnpm monorepo. Several issues currently prevent a clean CF Pages deployment: the build command in the docs uses npm (won't resolve pnpm catalog: deps), the post-build compression script creates .gz/.br files that CF Pages ignores (bloating upload), the .env has no production backend URL, cache headers are missing, and the Replit-specific vite plugin import can fail outside Replit.

Done looks like
pnpm install && pnpm run build succeeds cleanly inside artifacts/syrabit (verified by running the build)
The build produces a dist/ directory with no unnecessary .gz/.br files (CF Pages handles compression itself)
A _headers file exists in public/ with proper cache-control for static assets
The Replit vite plugin dynamic import is guarded so it doesn't break the build when @replit/vite-plugin-runtime-error-modal is absent
.env.example documents all required production env vars
docs/DEPLOYMENT.md is updated with correct CF Pages build settings (pnpm, not npm)
Out of scope
Cloudflare Workers / edge-proxy deployment (separate concern)
Backend deployment
Frontend feature changes
Domain/DNS configuration (manual in CF dashboard)
Setting actual env var values in CF dashboard (user does this manually)
Tasks
Remove or skip the compress-assets.mjs post-build step — Cloudflare Pages applies its own brotli/gzip at the edge, so the .gz/.br files just bloat the upload size without benefit.
Guard the Replit vite plugin import so it gracefully no-ops when the package isn't installed (CF Pages build environment won't have @replit/* packages). Currently the code checks REPL_ID env var but the dynamic await import() can still throw at build time if the package is missing — wrap it in a try/catch.
Add artifacts/syrabit/public/_headers with cache-control rules: immutable long-cache for hashed /assets/* files, short cache for index.html and service worker.
Update .env / .env.example with the production variable list (VITE_BACKEND_URL=https://api.syrabit.ai, VITE_WORKER_API_URL, VITE_GA_MEASUREMENT_ID, NODE_ENV=production).
Update docs/DEPLOYMENT.md — change the build command from npm install && npm run build to pnpm install && pnpm run build, and note that the CF Pages root directory should be set to artifacts/syrabit.
Run the build to verify it completes without errors and the dist/ output looks correct.
