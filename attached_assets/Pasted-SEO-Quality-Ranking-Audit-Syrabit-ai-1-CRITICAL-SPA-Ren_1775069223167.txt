SEO Quality & Ranking Audit — Syrabit.ai
1. CRITICAL: SPA Rendering — Your content is invisible to most search engines
This is the single biggest SEO problem. Syrabit is a client-side rendered SPA (React with Vite). There is no SSR, no pre-rendering, and no static generation.

Here's what happens when a search engine visits any page:

Googlebot can execute JavaScript, so it eventually sees your content — but with a "second wave" indexing delay (hours to days)
Bing, Yandex, DuckDuckGo, Baidu — limited or no JS rendering. They see a mostly blank index.html with only the static meta tags
AI crawlers (GPTBot, ClaudeBot, PerplexityBot) — no JS execution at all. Your content doesn't exist for them
You built an elaborate botRenderPlugin in the Vite config that serves full HTML to bots — but this only works in development mode (configureServer is dev-only in Vite). In production, Vite builds static files and this middleware doesn't exist. Your thousands of programmatic SEO pages are effectively invisible to most crawlers in production.

Impact: This likely explains poor ranking despite having thousands of content pages.

2. HIGH: Duplicate title template issue
The PageMeta component uses titleTemplate={"%s | Syrabit.ai"}. But the index.html already has a hardcoded <title> tag: "Syrabit.ai — Educational Browser For Assam Board Students".

On client-side navigation, react-helmet-async updates the title correctly. But on initial server load (what crawlers see first), the static title stays. This means:

Every page initially shows the same title to crawlers that don't run JS
Google may index duplicate titles across hundreds of pages
3. HIGH: Structured data quality issues
The JSON-LD implementation has several problems that could trigger Google Search Console warnings:

FAQPage schema is deprecated — Google discontinued FAQ rich results for most sites in August 2023. The effort spent generating FAQ schema produces no ranking benefit
Course schema misuse — Google's Course rich results require hasCourseInstance with courseSchedule fields. Using it for study notes without proper enrollment data is a schema mismatch that could trigger "unparsable structured data" warnings
Triple structured data on index.html — The static HTML has WebSite, EducationalOrganization, AND WebApplication schemas. The WebApplication includes an aggregateRating of 4.8 from 127 reviews — if these aren't real, verifiable reviews, this violates Google's structured data policies and could result in a manual action
Duplicate schemas — Both the static index.html and SeoTopicPage.jsx inject Article schemas. On the landing page, the static schemas and PageMeta's schemas can coexist, creating conflicting signals
4. HIGH: No internal linking from the SPA shell
The index.html (what non-JS crawlers see) contains:

Zero links to any content pages
No navigation menu
No sitemap link in the HTML (only in robots.txt)
This means for crawlers that can't execute JavaScript, there is no way to discover content through internal link crawling. They depend entirely on sitemap discovery.

5. MEDIUM: Sitemap architecture issues
The sitemap setup has redundancy problems:

Two sitemap indexes pointing to the same sub-sitemaps — robots.txt lists both /sitemap.xml (static file pointing to API sitemaps) AND /api/seo/sitemap-index.xml (API-generated index). Both reference the same child sitemaps. This confuses crawl budget
All 7 individual sub-sitemaps also listed separately in robots.txt — Google recommends listing only the index, not every child sitemap
