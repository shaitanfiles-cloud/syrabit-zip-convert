b) Edge worker patch — route /llms.txt, /llms-full.txt, /.well-known/ai-plugin.json to backend (fixes D3, D4)
Add to workers/edge-proxy/src/index.ts, in the same place as the existing /api/robots.txt proxy logic, BEFORE the SPA fallback:

// LLM/bot discovery files: proxy to backend, never to Pages SPA
const LLM_DISCOVERY_PATHS = new Set([
  "/llms.txt",
  "/llms-full.txt",
  "/.well-known/ai-plugin.json",
]);
if (LLM_DISCOVERY_PATHS.has(url.pathname)) {
  // Map to backend equivalents (note the /api/ prefix matches your backend mounts)
  const backendPath = url.pathname.startsWith("/.well-known/")
    ? url.pathname                       // backend handler is already at this exact path
    : `/api${url.pathname}`;             // /llms.txt → /api/llms.txt
  return fetch(`${env.BACKEND_URL}${backendPath}`, {
    headers: request.headers,
    cf: { cacheTtl: 86400, cacheEverything: true },
  });
}

(Caching values follow the existing CACHE_TTLS map convention in the worker.)
