Maximize Startup Credits — Full Provider Integration
What & Why
Integrate every startup-credit provider into Syrabit's provider stack via the Cloudflare AI Gateway BYOK system, and update llm.py's dispatch logic to follow the 80/20 rule: Workers AI handles 80% of each category for free; credit providers absorb the remaining 20% at peak load, saturation, or fallback. This maximises the value of ~$5k in entry-level credits (scalable to ~$400k) while keeping the marginal cost near zero at normal load.

Done looks like
Workers AI is the gated primary for chat, embeddings, reranking, translation, STT, vision, and safety — all free-tier calls.
Every credit provider listed below is wired through CF AI Gateway BYOK so no raw API keys live in Railway env vars.
llm.py select_provider / route_for_task applies load-based cascade: if Workers AI gateway metrics exceed 80% saturation, the next credit provider in the priority list is picked automatically.
Replicate is added as a new providers/replicate.py client (vision-gen fallback, $300 credits).
OpenRouter, Groq, Cerebras, Cohere, Pinecone AI, Cartesia, Deepgram, Baseten, and Sarvam AI all have healthy BYOK paths confirmed via the gateway smoke test.
The admin health panel shows per-provider credit burn rate, remaining quota estimate, and a "credits low" warning card for any provider below 20% estimated remaining.
A nightly cron logs per-provider call counts to the existing audit digest so spend is visible without leaving the admin panel.
The admin credit burn panel includes an AWS hosting row showing estimated App Runner spend vs. the $1k AWS Activate grant (~$21/month → ~47 months of free hosting).
A CloudWatch billing alarm is set at $800 so the team is alerted before credits run out.
Out of scope
Vertex AI and Gemini — removed entirely; no calls, no fallback, no gateway config.
AWS Bedrock or any AWS AI services — AWS credits cover hosting only.
Purchasing or topping up credits (manual step, outside code).
Switching vector DB from Pinecone to another provider.
Any frontend chat UI changes.
Workers AI model upgrades beyond what is already in the SLM slot table.
Steps
Add load-threshold constants and select_provider helper to llm.py — introduce a PROVIDER_PRIORITY map (one entry per feature: chat, embed, rerank, translate, tts, stt, vision, safety) with ordered fallback lists matching the 80/10/5/5 (or 80/20) splits. Implement select_provider(feature, lang) that reads cached CF gateway saturation metrics; if Workers AI load < 0.80 it returns workers_ai, otherwise it walks down the priority list. Remove any Vertex/Gemini entries from these lists. Thread the updated logic through route_for_task and _call_llm_raw.

Remove Vertex/Gemini from the codebase — delete or stub out all calls to vertex_chat.py and vertex_services.py so they are no longer invoked anywhere. Remove Gemini-related keys, model aliases (gemini-2.0-flash, gemini-2.5-flash), and gateway slugs from config.py and llm.py. Ensure no import or fallback path still references these modules.

Wire OpenRouter through CF AI Gateway BYOK — add _OPENROUTER_KEY BYOK header path in config.py (pattern already exists for Groq/Cerebras). Confirm get_provider_base_url('openrouter') resolves to the gateway slug. Update llm.py chat fallback list to include openrouter at position 2 (after workers_ai, before groq).

Wire Cohere embed through CF AI Gateway BYOK — confirm providers/cohere.py uses get_provider_base_url('cohere') and BYOK headers. Add a COHERE_EMBED_PRIMARY env-flag check so Assamese/Hindi precision embedding routes to Cohere when the flag is set, falling back to Workers AI bge-m3.

Wire Pinecone AI through CF AI Gateway BYOK — confirm providers/pinecone_ai.py uses the gateway slug. Route multilingual reranking calls through it at 20% of rerank volume (language ≠ English), with Workers AI bge-reranker-base as primary.

Wire Cartesia and Deepgram TTS through CF AI Gateway BYOK — confirm providers/cartesia.py paths use the gateway URL. Add Deepgram as providers/deepgram.py if not present, using the deepgram gateway slug. Split TTS dispatch: Cartesia (sonic-2) for <150ms latency targets; Deepgram (Aura-2) for longer-form audio; Workers AI whisper for STT.

Wire Baseten through CF AI Gateway BYOK — confirm providers/baseten.py uses the OpenAI-compatible gateway path. Ensure fine-tuned model names are listed in the SLM slot table so route_for_task('content') can select Baseten when available.

Add providers/replicate.py — new client for Replicate's REST API routed through the replicate CF AI Gateway slug. Use for vision-generation overflow at 20% of vision calls when Workers AI llama-3.2-11b-vision is saturated.

Confirm Sarvam AI BYOK path — ensure all three Sarvam LLM keys (_SARVAM_LLM_KEY, _SARVAM_LLM_KEY_2, _SARVAM_LLM_KEY_3) rotate correctly through the gateway. The _is_indic_lang('as') check in llm.py already routes Assamese to Sarvam; confirm this bypasses Workers AI entirely for that language.

Add credit-burn monitoring to admin health panel — add a ProviderCreditCard component (frontend) that polls a new /api/admin/provider-credits endpoint. The endpoint reads per-provider call counters from the existing Redis/KV store and returns estimated spend vs. credit grant for each provider. Show a "credits low" warning at <20% remaining. Include this in the nightly audit digest sent to Slack.

Tag App Runner with cost allocation tag — add the project: syrabit tag to the App Runner service (account 926046660612) so the AWS billing dashboard attributes hosting spend correctly and the $1k Activate grant is tracked in isolation from any other AWS usage.

Set a CloudWatch billing alarm at $800 — create a billing alarm in us-east-1 (the only region that surfaces account-level billing metrics) that fires to the existing ops Slack channel when cumulative AWS charges cross $800. This leaves a $200 buffer before the $1k Activate credit is exhausted, giving time to switch back to Railway if needed. Document the math in the alarm description: App Runner (1 vCPU / 2 GB) ≈ $21/month → $1k ≈ 47 months of free hosting.

Add AWS hosting row to the admin credit burn panel — extend the /api/admin/provider-credits endpoint and the frontend ProviderCreditCard component (from step 10) to include an AWS row. Pull the current month's App Runner spend from the AWS Cost Explorer API and display it against the $1k grant. Show a "credits low" warning when estimated remaining drops below $200.

Gateway smoke test — extend the existing nightly smoke run to hit each provider's gateway slug with a minimal test call and confirm the BYOK header path returns 200. Log results per provider so broken BYOK configs surface before they affect real users.
