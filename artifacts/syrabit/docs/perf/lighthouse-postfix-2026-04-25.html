What changed

  1. SPA fallback HTMLs (/, /about, /pricing, /login, /signup, …) now inline ~7–16 KB of above-the-fold critical CSS in <head> and rewrite the <link rel="stylesheet"> into rel="alternate stylesheet preload" … onload="this.rel='stylesheet'" + a <noscript> fallback link.
  2. Prerendered SEO routes (/library, /browser) previously shipped the entire 141 KB stylesheet inline. The new step un-wraps that giant <style data-inline-css>, runs Beasties on the prerendered SSR markup so it extracts the subject-card styles that are actually visible, and emits the same preload+swap pattern.
  3. A build-time postcondition in scripts/verify-all.mjs fails the build if any of / + REQUIRED_CANONICAL_ROUTES + /library + /browser ships a render-blocking link to the main bundle outside <noscript> (15 routes gated).
